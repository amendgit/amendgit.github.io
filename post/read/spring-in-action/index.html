5.4 Processing Forms

Form提交到数据，主要在Controller的方法的参数声明对应的model就可以了，Spring可以自动放到model中。例如：将Form中的fistName、lastName、username和password数据放到Spitter的firstName、lastName、username和password成员中。

@Controller
@RequestMapping("/spitter")
class SpitterController {
	@RequestMapping(value="/register", method=GET)
	void register(Spitter spitter) {
		...
	}
}

为了防止用户数据反复提交，注册的方法返回时，用了return "redirect:/spitter/username"，这样InternalResouceReslover会将这个请求重定向到用户的Profile页。同时，还支持forward:前缀。

Spring提供了对Java Validation API的支持，只需要保证存在Java API的实现，如Hibernate Validator即可，不需要额外的配置工作。

JVA的注解包括：

  • @AssertFalse
  • @AssertTrue
  • @DecimalMax
  • @DecimalMin
  • @Digits
  • @Future
  • @Max
  • @Min
  • @NotNull
  • @Null
  • @Past
  • @Pattern
  • @Size
public class Spitter {
	private Long id;

	@NotNull
	@Size(min=5, max=16)
	private String username;

	@NotNull
	@Size(min=5, max=25)
	private String password;

	@NotNull
	@Size(min=2, max=30)
	private String firstName;

	@NotNull
	@Size(min=2, max=30)
	private String lastName;

	...
}

使用这些注解进行验证

@RequestMapping(value="/register", method=POST)
public String processRegsteration(
	@Valid Spitter spitter, 
	Errors errors) {
	if (errors.hasErrors()) {
		return "registerForm";
	}
	...
}

参数spitter需要带上@Valid注解，如果校验失败会将错误放到errors中。注意errors要紧跟这被校验的参数转换。

6. Rendering Web views

6.1 Understanding view resolution

将请求处理的逻辑解耦到controllers和views是Spring MVC的一个重要feature。

注意view需要和cotroller就model的内容达成共识。

从view的logical view name到对应的view实现，是view resolvers的职责所在。

ViewResolver是一个接口

public interface ViewResolver {
	View resolveViewName(String viewName, Locale locale) throws Exception;
}

public interface View {
	String getContentType();
	void render(Map<String, ?> model, HttpServletRequest request, HttpServletResponse response) throws Exception;
}

Spring提供了一些拆箱即用的实现

  • BeanNameViewResolver
  • ContentNegotiatingViewResolver
  • FreeMarkerViewResolver
  • InteralResourceViewResolver
  • JasperReportsViewResolver
  • ResourceBundleViewResolver
  • TilesViewResolver
  • UrlBasedViewResolver
  • VelocityLayoutViewResolver
  • VelocityViewResolver
  • XmlViewResolver
  • XsltViewResolver

思考：这样的方式应该是为请求响应是的web应用设计的。而像React这样的单页web应用，是不会用到这个的。现在的React应 用应该是用不到这些Spring功能的。反而更像是native的方式，自己管理UI，然后需要数据了就发起请求到服务端。获取数据 后进行渲染。

6.2 Creating JSP Views

  • 可以创建一个叫viewResolver的bean，Spring会使用这个id去今夕view resolver
  • 使用InternalResourceViewResolver可以设置jsp文件的前缀和后缀，然后设置view的基类，比如JSTLView，这样就能支持tag了。
  • tags的库有两种：1. 支持form绑定的tag库 2. 辅助form创建的各种各样工具
  • form绑定有多种定义的tag支持：sf:form sf:input sf:password等等
  • sf:form的commandName属性用于指定相应的model，之后就可以在子节点中用path去指定属性名。
  • sf:errors标签用于向用户提示错误。可以单个标签的显示，也可以集中在一起显示。
  • cssClass或cssErrorClass可以指定不同的error样式。
